## Описание
Проект содержит тесты доменных моделей SpecFlowTests.Domain

## Особенности проекта
Для комфортной работы с тестами Specflow рекомендуется установить расширение для Visual Studio https://marketplace.visualstudio.com/items?itemName=TechTalkSpecFlowTeam.SpecFlowForVisualStudio2022
Для среды разработки Rider так же существует плагин.

Настройка проекта осуществляется путём изменения файла `specflow.json`. Для корректной работы русского языка, нужно установить значение `ru-RU` для двух свойств - `bindingCulture.name` и `language.feature`.

Файлы со сценариями находятся в папке `Features` и имеют расширение `.feature`.
Для каждого куска функциональности лучше создавать отдельный `.feature` файл, который будет содержать сценарии только для этой функциональности.

В каждом сценарии должен быть **один и только один** блок `Когда`. Это важно, т.к. Specflow сценарий должен тестировать **одно конкретное** действие пользователя. Иначе не будет понятно, какое действие в сценарии является целевым.
